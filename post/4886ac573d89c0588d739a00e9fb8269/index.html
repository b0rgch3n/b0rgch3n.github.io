安卓虚拟机

虽然 Android 平台使用 Java 语言进行开发，但 Android 程序不是运行在标准的 Java 虚拟机，而是运行在 Dalvik 虚拟机。
Android 4.4 新增 Art 虚拟机，而 Android 5.0 后全面启用 Art。

Dalvik 虚拟机特点：

  • 占用内存小
  • 专有 DEX 可执行文件格式，体积小执行速度快
  • 常量池 32 位索引，寻址更快
  • 基于寄存器架构，有一套完整的指令系统
  • 提供了对象生命周期管理、堆栈管理、线程管理、安全和异常管理、垃圾回收等重要功能
  • 每个 Android 程序的进程都对应一个 Dalvik 虚拟机实例

与 Java 虚拟机的区别：

  • 运行内容不同：Java 虚拟机运行 Java 字节码；Dalvik 虚拟机运行 Dalvik 字节码
  • 可执行文件体积更小：Android SDK 的 dx 工具将 Java 字节码转换为 Dalvik 字节码，并消除所有冗余信息
  • 结构不同：Java 虚拟机基于栈架构，频繁出入栈，需要更多指令分派与内存访问；Dalvik 虚拟机基于寄存器架构，数据访问通过寄存器直接传递，数据访问速度更快

Dalvik 虚拟机执行程序的流程

  1. Android 系统启动先加载内核，内核加载后执行 init 进程，进行设备初始化，然后读取 inic.rc 并启动 Zygote 进程
  2. Zygote 进程初始化 Dalvik 虚拟机
  3. Zygote 进程执行 forkSystemServer() 方法，启动 system_server 进程，然后自身进入 Zygote 模式（Zygote 进程和 system_server 进程都通过 socket 等待命令）
  4. 当 Android 应用程序执行，同时 socket 发送执行请求到 system_server 进程，system_server 进程继而转发至 Zygote 进程
  5. Zygote 进程收到命令后执行 forkAndSpecialize() 方法 fork 自身创建一个 Dalvik VM 执行应用程序的入口函数


（流程图整理参考自《Android 软件安全与逆向分析》图3-5）

Dalvik 虚拟机的 JIT

Android 2.2 版本 Dalvik 虚拟机引入了 JIT 技术。
JIT 包含 2 种字节码编码方式：

  • method：以函数或者方法为单位进行编译。
  • trace：获取执行较为频繁的代码路径进行编译。

（Dalvik 默认使用 trace 方式进行代码编译）

Dalvik 汇编基础

Dalvik 寄存器
Dalvik 虚拟机基于寄存器架构，在 ARM 架构的 CPU 上运行。
Dalvik 将部分寄存器映射到 ARM 寄存器上，还有一部分通过调用栈进行模拟。
Dalvik 中用到的所有寄存器都是 32 位，支持任何类型，64 位则用 2 个相邻寄存器表示。
Dalvik 虚拟机支持 65536 个寄存器，寄存器采用 v0 作起始值，取值范围是 v0~v65535。

寄存器有 2 种不同的表示方法 —— v 命名法与 p 命名法
v 命名法所有函数中的局部变量和参数均以 v 开头的寄存器命名表示，从 v0 开始，依次递增。
p 命名法对函数局部变量命名不影响，只变化函数传参部分，函数中参数从 p0 开始，依次递增。
例如有一个 Hello 对象的 foo() 方法使用到 5 个寄存器，但该方法只有 2 个显式参数：
使用 v 命名法时寄存器分配 v0、v1 表示局部变量，v2 表示被隐式传入的 Hello 对象引用，v3、v4 表示方法的显式参数。
使用 p 命名法时寄存器分配 v0、v1 表示局部变量，p0 表示被隐式传入的 Hello 对象引用，p1、p2 表示方法的显式参数。
