Android 开发学习小记（持续更新中...）

2020-01-13

近来出于兴趣开始了 Android 逆向分析的学习，但由于对 Android 开发一片空白，于是也打算初略的过一下开发基础，于是便有了这篇小记。

小记主要内容来自于学习《Android 第一行代码（第二版）》、《Android 软件安全与逆向分析》以及“漏洞银行 - 逆向工程系列讲座”的过程。

小记暂时可能较为零碎，都是一些知识的整理，目前只为辅助我个人学习记忆，待有空会再加以体系化整理。

Android 程序生成大致步骤
1.打包资源文件，生成 R.java 文件
2.处理 aidl 文件，生成相应 Java 文件
3.编译工程源代码，生成相应的 class 文件
4.转换所有 class 文件，生成 dex 文件
5.打包生成 APK 文件
6.对 APK 文件进行签名
7.最后对签名后的 APK 文件进行字节码对齐处理

文章里提到的打包过程第一步的我个人觉得可以补充得再详细一些
res/raw 与 assets 的不同点
1.访问方式：res/raw 中的文件被访问时直接使用资源的 ID 即 R.id.filename；assets 中的文件不会被映射到 R.java，需要通过 AssetManager 类进行访问
2.目录结构：res/raw 下不允许又目录结构；assets 下可以又目录结构
3.读取文件资源方式不同，如下：

// 读取 res/raw 文件资源
InuptStream is = getSources().openRawResource(R.id.filename);
// 读取 assets 文件资源
InuptStream is =getAssets().open("filename");

// 注意：（该注意点来自“漏洞银行 - 逆向工程系列讲座”，个人暂未验证）
// 单个 assets 文件资源不能超过 1 MB，否则读取时会有异常抛出
// 但 res/raw 文件资源的读取没有大小限制
